# 📚 Modelos de Desenvolvimento de Software
## 🔹 1. Modelo Cascata (Waterfall)

O modelo cascata é um dos modelos mais tradicionais de desenvolvimento de software.

Ele segue uma abordagem **sequencial**, onde cada etapa deve ser concluída antes da próxima começar.

### 📌 Etapas

1. Levantamento de requisitos  
2. Projeto (design)  
3. Implementação (código)  
4. Testes  
5. Implantação  
6. Manutenção  

### ⚙️ Características

- Processo **linear e estruturado**
- Pouca flexibilidade para mudanças
- Cada fase depende da anterior

### ✅ Quando usar

- Requisitos **bem definidos e estáveis**
- Projetos pequenos ou previsíveis
- Sistemas críticos (ex: governo, bancos)

---

## 🔹 2. Modelo de Reuso (Baseado em Componentes)

O modelo de reuso consiste em desenvolver software utilizando **componentes já existentes**, ao invés de criar tudo do zero.

Esses componentes podem ser:

- bibliotecas
- frameworks
- APIs
- módulos prontos

### 📌 Ideia principal

> Reutilizar soluções já prontas para acelerar o desenvolvimento.

### ⚙️ Características

- Redução de tempo de desenvolvimento
- Menor custo
- Dependência de componentes externos
- Necessidade de adaptação

### ✅ Quando usar

- Sistemas que podem aproveitar soluções existentes
- Desenvolvimento rápido (MVPs)
- Projetos com uso de frameworks (React, Spring, etc.)

---

## 🔹 3. Modelo Ágil

O modelo ágil é uma abordagem moderna baseada em **iterações curtas e entregas contínuas**.

O software é desenvolvido em ciclos pequenos, com foco em **feedback constante do cliente**.

### 📌 Exemplos de metodologias ágeis

- Scrum  
- Kanban  
- XP (Extreme Programming)

### ⚙️ Características

- Alta flexibilidade
- Mudanças são bem-vindas
- Entregas frequentes
- Forte interação com o cliente

### ✅ Quando usar

- Requisitos **mudam com frequência**
- Startups e produtos digitais
- Sistemas em evolução constante

---

# 🔄 Diferença entre os Modelos

| Característica        | Cascata              | Reuso                      | Ágil                         |
|----------------------|---------------------|-----------------------------|------------------------------|
| Processo             | Sequencial          | Baseado em componentes      | Iterativo / incremental      |
| Flexibilidade        | Baixa               | Média                       | Alta                         |
| Mudanças             | Difíceis            | Moderadas                   | Fáceis                       |
| Velocidade           | Lenta               | Rápida                      | Rápida                       |
| Foco                 | Planejamento        | Reaproveitamento            | Cliente e adaptação          |
| Entregas             | Apenas no final     | Depende dos componentes     | Contínuas                    |

---

### Resumo Final:

- **Cascata:** modelo linear e sequencial, ideal para projetos com requisitos bem definidos.  
- **Reuso:** foca na reutilização de componentes existentes para reduzir tempo e custo.  
- **Ágil:** modelo iterativo com entregas rápidas e foco em adaptação e feedback do cliente.
